## Context

在現有的英文打字測試基礎上，擴展支援中文打字練習。使用者可能使用各種中文輸入法（注音、倉頡、無蝦米、拼音），需要一個查詢編碼的功能來幫助學習。

## Goals / Non-Goals

**Goals:**
- 支援中文文章打字練習
- 提供按 Tab 查詢編碼的功能
- 同時顯示四種輸入法編碼（注音、倉頡、無蝦米、拼音）
- 保持簡單，無需伺服器

**Non-Goals:**
- 完整的中文字典（只需常用字）
- 輸入法教學或互動式學習
- 使用者自訂文章功能
- 編碼資料的即時更新

## Decisions

### 編碼資料來源
使用內嵌的 JavaScript 物件儲存常用字編碼資料。這樣可以離線使用，不需要伺服器或外部 API。

**Alternatives considered:**
- 外部 API 查詢：需要網路，增加延遲
- 外部 JSON 檔案：需要額外請求，可能有 CORS 問題

### 編碼資料範圍
初期只包含最常用的 500-1000 個中文字的編碼資料。這些字覆蓋大部分日常文章。

**Alternatives considered:**
- 完整 Unicode 中文字：檔案太大，不實際
- 只包含文章中的字：每次新增文章需更新資料

### Tab 鍵行為
按 Tab 鍵顯示編碼提示，不觸發瀏覽器預設的 Tab 行為（切換焦點）。使用 `e.preventDefault()` 阻止預設行為。

**Alternatives considered:**
- 使用其他按鍵（如 F1）：Tab 更直覺
- 滑鼠懸停顯示：觸控裝置不方便

### 提示框顯示方式
使用固定定位（`position: fixed`）在右下角顯示，不影響打字區域的佈局。

**Alternatives considered:**
- 彈出對話框：太干擾
- 內嵌在頁面中：佔用空間

## Risks / Trade-offs

**[Risk] 編碼資料不完整** → 使用者查詢生僻字時可能無資料。顯示「無資料」提示，未來可擴充。

**[Risk] 無蝦米編碼版權** → 無蝦米是商業輸入法，編碼可能有版權問題。初期可先支援其他三種，無蝦米標註為「參考用」。

**[Trade-off] 資料檔案大小** → 500-1000 字的編碼資料約 50-100KB，可接受。
